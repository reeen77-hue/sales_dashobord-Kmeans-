📦 商品クラスタリング（KMeans）
目的

売上データをもとに、商品を性質ごとにグループ化（クラスタリング）し、
在庫・売場・販促などの実務アクションにつなげる示唆を得ることを目的としています。

本分析では 正解ラベルを持たないデータに対して、
特徴量設計 → クラスタリング → 解釈、という流れを重視しました。

使用手法

特徴量作成（商品単位）

平均日次売上金額

平均日次販売数量

客単価（AOV）

1取引あたり点数（UPT）

標準化（StandardScaler）

KMeans（K=3）

PCAによる2次元可視化（解釈補助）

クラスタ可視化（PCA）

※ 高次元の特徴量をPCAで2次元に圧縮し、
クラスタの分離傾向を視覚的に確認しています。
解釈自体は、元の特徴量の集計値をもとに行います。

クラスタ別の解釈と実務アクション
🟢 Cluster 1：主力・定番商品（例：501 / 502）

特徴

平均日次売上・数量ともに高い

売上が安定している

店舗売上のベースを構成

解釈

日常的に回転し、安定して売上を作る「定番商品群」

実務アクション

欠品を最優先で防止

常に売場の中心に配置

在庫はやや厚めに確保

🔵 Cluster 0：高単価・訴求型商品（例：511）

特徴

数量は多くないが、客単価が高い

1点あたりの価値が高い

解釈

接客やスタイリング提案によって選ばれる「価値訴求型商品」

実務アクション

接客強化・コーディネート提案

POPやビジュアルでの価値訴求

無理な在庫積み増しは避ける

🟠 Cluster 2：準定番・サブ商品（例：505）

特徴

売上・数量ともに中程度

条件次第で伸びる可能性

解釈

主力ではないが、売場やタイミング次第で貢献度が変わる商品

実務アクション

セール時や主力商品の横で展開

売場変更・施策テストの対象

データを見ながら役割を再定義

まとめ

本クラスタリングでは、
「どの商品が似ているか」を自動的に分けた上で、
その結果を人間が業務目線で解釈することを重視しました。

クラスタリング自体に正解はありませんが、

商品の役割整理

在庫配分

売場戦略
といった実務判断の補助として、有効な示唆が得られました。
